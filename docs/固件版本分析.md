## WebSocket改造策略

### 改造目标
将串口通信（16字节二进制协议）替换为WiFi WebSocket通信（JSON格式指令）。

### 保留的核心组件
- `HLSCL` - 舵机通信库
- `Homing` - 归位模块
- `TaskSyncRead_Core1` - 核心1同步读取任务
- 软限位和热保护功能

### 替换的组件
- `Serial` 通信 → `WiFi` + `WebSocketsServer`
- 二进制协议 → JSON协议

### 新增的功能
- WiFi AP/STA模式
- WebSocket服务器
- JSON命令解析

## 协议转换

### 原始串口协议（16字节二进制）

| 字节 | 内容 |
|------|------|
| 0 | 操作码 (0x01-0x32) |
| 1-14 | 参数数据 |
| 15 | 保留 |

### WebSocket协议（JSON）

```json
// 单关节控制
{"type": "joint_control", "data": {"joint_id": "index_proximal", "angle": 45.0, "duration_ms": 500}}

// 多关节控制
{"type": "multi_joint_control", "data": {"joints": [{"joint_id": "thumb_proximal", "angle": 30.0}], "duration_ms": 500}}

// 归位
{"type": "homing"}

// 获取状态
{"type": "get_states"}
```
